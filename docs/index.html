Compiler for a subset of the C language

2024-01-01

This project was made as part of a course about Compilers and Execution Environments. The task was to generate assembly code for a given string of c code. I implemented parsing, typechecking, AST to IR and IR to assembly. My compiler was able to support all binary operators, loops and conditionals. The compiler was implemented in scala using pattern matching and recursion.

This is probably the most difficult project I have ever made so far. The code base was around 1000 lines of code of complex complier logic. The performance of the generated assembly was far from great. For any expression I would save the values on the stack and only if necessary use registers. This meant that a lot of simple would use stack allocation even though enough registers would be available. Overall it was a very challenging project but very rewarding in the end.

Scala x86 Functional Programming Parsing

Building and setting up an emulated employee network

2024-01-03

This project was made as part of a course called Building Network Security. We were given two routers and a few ethernet cables and we had to setup a company that would have two branches: one in Stockholm and one in London. The fictive employees needed access to a secure fileserver in both Stockholm and London that should be accessible both from the company network and from using the VPN.

In this project I learned how to configure many linux systems and how to debug network issues of different sorts. I have configured an OpenWRT router with firewall rules, wireguard vpn and wpa3 wifi security. I have also deployed a freeradius server for wpa3 eap-tls authentication. To monitor the network we configured snort with fail2ban to block ban the IPs of malicious actors.
For the secure file servers we deployed two OwnCloud servers that had authentication integrated with an LDAP server deployed on a FreeIPA Fedora VM.

Mail Man strikes again Demo

2019-03-01

3D game developed in Unity for a gamejam with the theme "UnexpectedGame".Because I did not have a team I have joined a team on discord with people that I did not know. I worked with two students from USA and someone else from UK. Because we the gamejam spanned two weeks we had a lot of time to try and prototype many different ideas. Unfortunately, we were too optimistic about our skills and we had to modify the initial idea of the game to something that we could implement in the given time. The idea we implemented is that the player is a mailman that has to deliver mail to the right boxes. The player can throw the mail to deliver it to the matching mail box. To make the game more challenging we added a timer for each level. In higher levels, boxes drop from the "sky" that block you from throwing the mail. We have added more crazy ideas such as dynamite for destroying the boxes. You can see the game submission at the url on top. During this experience I have learned how to communicate in a team and how to be clear about my expectations from the other team members. Unfortunately, half way through the project, one team member was disappointed by the fact that we limit the scope of the game too much and decided to leave the team. However, we handled the situation correctly and we continued the project without any hard feelings. I have also learned how to structure the code better and it was my first project on Github where I would have another contributor apart from myself (haha). It was this project that taught me how to solve merge conflicts, create branches and work with git in a more proficient way.
